# Simplify Rock Pattern to Single Option

## Problem

Having two rock patterns (Mountain/Rock and Grey Rock) is confusing. User edits one but the other gets applied.

## Solution

Keep only ONE rock pattern - rename to "Rock" and remove the distinction.

### 1. Update Biome Dropdown in UI

In [terrain_generator_demo.html](terrain_generator_demo.html), change the dropdown options (around line 718):

**Before:**

```html
<option value="rock_mountain">Mountain/Rock</option>
<option value="rock">Grey Rock</option>
```

**After:**

```html
<option value="rock">Rock</option>
```

### 2. Simplify Pattern Selection Logic

In [js/data/uoTileSetsClean.js](js/data/uoTileSetsClean.js), update `getCleanTileAtPosition()` and `getContextualTile()` to only check `rock` pattern:

```javascript
if (biome === 'rock') {
    const patternTile = getBlobPatternTile('rock', x, y);
    if (patternTile) return patternTile;
}
```

### 3. Update Tile Palette Ranges

In `terrain_generator_demo.html`, update `BIOME_TILE_RANGES` to combine mountain rock and grey rock tiles under single `rock` key:

```javascript
'rock': [
    { start: 0x01D3, end: 0x01DA, name: 'Mountain Rock' },
    { start: 0x00E4, end: 0x00E7, name: 'Grey Rock 1' },
    { start: 0x00F4, end: 0x00F7, name: 'Grey Rock 2' },
    // ... other grey rock ranges
],
```

(Remove the separate `rock_mountain` entry)

### 4. Clean Up localStorage

Add cleanup code to remove old `rock_mountain` pattern from localStorage on page load.

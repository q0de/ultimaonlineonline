# Show Stamp Configuration Name on Labels

## Problem

Currently labels show individual tile positions (NW, N, NE, etc.) but not which overall stamp config is being used (North, South, Corner NE, Inner SW, etc.).

## Solution

Pass the pattern's `directionId` through to tiles and display it on the label.

### 1. Store directionId on tiles when stamp is placed

In [js/data/waterTileMappings.js](js/data/waterTileMappings.js), in the `placeStamp()` function around line 664, add:

```javascript
targetTile.stampConfigName = pattern.directionId;  // "north", "corner_ne", etc.
```

### 2. Update label display to show config name

In [terrain_generator_demo.html](terrain_generator_demo.html), in the `renderPatternLabels()` function, change the label from showing position to showing the stamp config name:

```javascript
// Instead of: tile.terrainEdgePosition (NW, N, NE...)
// Show: tile.stampConfigName (north, corner_ne, etc.)
const configLabel = (tile.stampConfigName || '').toUpperCase().replace('_', ' ');
```

### 3. Optional: Show config name only at stamp CENTER

To reduce clutter, only show the config name on the CENTER tile of each stamp, while keeping position labels on other tiles:

```javascript
if (tile.terrainEdgePosition === 'CENTER') {
    label = tile.stampConfigName.toUpperCase();  // "NORTH", "CORNER NE"
} else {
    label = tile.terrainEdgePosition;  // "NW", "N", etc.
}
```

## Result

Each stamp group will show which configuration it's using (NORTH, SOUTH, CORNER NE, INNER SW, etc.) making it clear which stamp from your configurator is being applied.
